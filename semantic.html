Semantic HTML

What it means · Why it matters · How to use it

01
What is Semantic HTML?
Definition
Semantic HTML means using HTML elements that carry meaning about the structure and content of a web page — not just its appearance. Instead of using generic <div> and <span> tags for everything, semantic elements like <header>, <main>, <article>, and <footer> clearly describe what that content is and where it belongs on the page.
02
Common Semantic Elements
Key tags to know
  • <header>
    Top section of a page or section
  • <nav>
    Navigation links
  • <main>
    Primary content of the page
  • <article>
    Self-contained, reusable content
  • <section>
    Thematic grouping of content
  • <footer>
    Bottom of page or section
03
Why Use Semantic HTML?
3 main reasons
  • Accessibility
    Screen readers understand the page structure better
  • SEO
    Search engines rank pages with clear structure higher
  • Maintainability
    Easier to read and edit code for developers
04
Code Example
Semantic vs non-semantic
❌ Without Semantics
<div class="header">
  <div class="nav">...</div>
</div>
<div class="content">
  <div class="article">...</div>
</div>
<div class="footer">...</div>
✅ With Semantics
<header>
  <nav>...</nav>
</header>
<main>
  <article>...</article>
</main>
<footer>...</footer>
05
Accessibility
Helping all users
Screen readers and assistive technologies rely on semantic HTML to navigate pages. When you use <nav>, users can jump straight to navigation. When you use <main>, they can skip to the core content immediately. Semantic markup makes your site usable by everyone.
06
SEO Benefits
Search engine visibility
Search engines like Google crawl your HTML to understand what a page is about. Semantic tags tell the crawler: "this is the main content", "this is a navigation menu", and "this is a self-contained article." Clear structure leads to better indexing and potentially higher rankings.
